================================================================================
TECHNICAL IMPLEMENTATION
================================================================================

ALGORITHM: Map Generation
1. Parse AWS snapshot JSON
2. Extract VPCs, subnets, security groups, EKS, LBs, RDS
3. Generate rooms with proper dimensions
4. Position resources hierarchically
5. Create wall boundaries
6. Apply security (locked/unlocked)
7. Assign colors by resource type
8. Connect with hallways

ALGORITHM: 3D Rendering
1. Cast 120 rays from player position
2. For each ray:
   - Test intersection with all walls
   - Find closest wall hit
   - Calculate distance
   - Apply fish-eye correction
   - Compute wall height (perspective)
   - Apply distance shading
   - Draw vertical wall slice
3. Render floor and ceiling
4. Draw HUD and minimap

PERFORMANCE:
- Optimized for 10-50 AWS resources
- Resource limits prevent clutter
- 60 FPS on modern hardware
- Efficient collision detection
- Minimal memory footprint

HOW TO USE
================================================================================

1. PREPARATION:
   Ensure you have AWS snapshots in:
   ~/AWS_Architecture_explorer/snapshots/*.json

   If not, create one:
   cd ~/AWS_Architecture_explorer
   python collect_snapshot.py --region us-west-2 --out snapshots/snapshot.json

2. INSTALLATION:
   cd ~/AWS_Architecture_explorer/AWS_DOOM/game
   pip install pygame>=2.5.0

3. RUN THE GAME:
   ./run_aws_doom.sh          (Linux/Mac)
   run_aws_doom.bat           (Windows)
   python3 aws_doom.py        (Manual)

4. PLAY:
   - Use W/A/S/D to navigate
   - Press E to collect keys
   - Press M to toggle map
   - Press ESC to quit
