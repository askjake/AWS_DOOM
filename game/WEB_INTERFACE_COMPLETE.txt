║                          HOW THE WEB INTERFACE WORKS                          ║
╚══════════════════════════════════════════════════════════════════════════════╝

ARCHITECTURE:

    ┌─────────────────┐                 ┌──────────────────┐
    │  Web Browser    │                 │   Flask Server   │
    │  (Your Device)  │                 │   (Port 5000)    │
    │                 │                 │                  │
    │  - HTML/CSS/JS  │◄───WebSocket──►│  - Game Loop     │
    │  - Display Game │                 │  - Frame Render  │
    │  - Send Keys    │                 │  - Input Handler │
    └─────────────────┘                 └────────┬─────────┘
                                                  │
                                         ┌────────▼─────────┐
                                         │  pygame (Headless│
                                         │  Game Engine)    │
                                         │  - AWS Map       │
                                         │  - Raycasting    │
                                         └──────────────────┘

FLOW:
1. Server runs pygame in headless mode (no window)
2. Game engine renders frames to offscreen surface
3. Frames converted to PNG images
4. Images encoded as base64 strings
5. Sent to browser via WebSocket (Socket.IO)
6. Browser displays image and sends keyboard inputs back
7. Loop repeats at 30 FPS

  → Click on game canvas to focus
  → Check browser console for errors
  → Try refreshing page

"Low FPS / Lag"
  → Reduce frame_rate in aws_doom_web.py
  → Use wired network connection
  → Close other applications
  → Try from same machine (localhost)

"Connection keeps dropping"
  → Check network stability
  → Verify firewall not blocking WebSocket
  → Check server logs for errors
