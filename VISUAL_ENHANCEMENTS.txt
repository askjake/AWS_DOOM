╔══════════════════════════════════════════════════════════════════════════════╗
║                          2. WALL TEXTURES                                    ║
╚══════════════════════════════════════════════════════════════════════════════╝

Each AWS resource type has a unique texture pattern:

  Resource Type        Texture Pattern    Visual
  ──────────────────   ───────────────    ──────────────────
  VPC                  Grid (3)           ▦ Crosshatch grid
  Subnet               Stripes (1)        ║ Vertical lines
  Security Group       Dotted (2)         ∴ Dot pattern
  EKS Cluster          Grid (3)           ▦ Crosshatch grid
  Load Balancer        Stripes (1)        ║ Vertical lines
  RDS Database         Dotted (2)         ∴ Dot pattern
  Hallway              Solid (0)          █ Plain color

TEXTURE RENDERING:

  Pattern 0 (Solid):
    - Plain colored wall
    - No additional patterns
    - Used for hallways

  Pattern 1 (Vertical Stripes):
    - Alternating light/dark vertical bands
    - Stripe width: wall_width / 8
    - 20% darker stripes
    - Creates depth perception

  Pattern 2 (Dotted):
    - Circular dots arranged vertically
    - Dot spacing: wall_height / 10
    - 20% lighter than base color
    - Centered on wall

  Pattern 3 (Grid):
    - Horizontal lines at regular intervals
    - Grid size: wall_height / 8
    - 30% darker than base color
    - Creates structural appearance

All patterns apply distance shading (darker when farther away).

╔══════════════════════════════════════════════════════════════════════════════╗
║                          3. WALL SIGNS                                       ║
╚══════════════════════════════════════════════════════════════════════════════╝

Signs display resource information directly on walls:

SIGN APPEARANCE:
  ┌─────────────────────────────┐
  │  [Semi-transparent black]   │
  │                             │
  │    VPC 10.0.0.0/16          │ ← Resource name (yellow)
  │                             │
  │    → Subnets                │ ← Direction (green)
  │                             │
  └─────────────────────────────┘

SIGN COMPONENTS:

  1. Background:
     - Semi-transparent black (180 alpha)
     - Height: 1/3 of wall height (max 60px)
     - Centered vertically on wall
     - Creates contrast for text

  2. Main Text (Yellow):
     - Resource name truncated to 20 chars
     - Font: Small font (18px)
     - Color: (255, 255, 100) bright yellow
     - Positioned in upper part of sign

  3. Direction Text (Green):
     - Shows where this passage leads
     - Font: Small font (18px)
     - Color: (100, 255, 100) bright green
     - Positioned in lower part of sign
     - Includes arrow symbols (→ ← ↑ ↓)

RENDERING RULES:

  - Only render signs on walls within 200 pixels
  - Only show on center-facing walls (±10 rays from center)
  - Prevents text clutter at edges and distant walls
  - Ensures readability

SIGN CONTENT BY RESOURCE:

  VPCs:          "VPC 10.0.0.0/16"     + "→ Subnets"
  Subnets:       "Subnet 10.0.1.0/24"  + direction
  Security Grps: "SG web-servers"      + status
  Load Balancer: "LB main-alb"         + "→ VPC vpc-123"
  Hallways:      "CORRIDOR"            + "→ East" / "↑ North"
  RDS:           "RDS prod-db"         + engine type
  EKS:           "EKS prod-cluster"    + status

╚══════════════════════════════════════════════════════════════════════════════╝

Visual arrows guide navigation:

ARROW TYPES:

  → Right Arrow:   Points east / to the right
  ← Left Arrow:    Points west / to the left
  ↑ Up Arrow:      Points north / forward
  ↓ Down Arrow:    Points south / backward

RENDERING:

  - Triangle polygons drawn on wall
  - Color: Bright green (100, 255, 100)
  - Size: 20px (or 1/4 wall height)
  - Position: Center of wall vertically
  - Only shown when direction text present

DIRECTION CALCULATION (Hallways):

  Angle Range          Direction    Arrow
  ─────────────────    ─────────    ─────
  -π/4 to π/4          East         →
  π/4 to 3π/4          South        ↓
  3π/4 to -3π/4        West         ←
  -3π/4 to -π/4        North        ↑

╔══════════════════════════════════════════════════════════════════════════════╗
║                          7. NEW RENDERER METHODS                             ║
╚══════════════════════════════════════════════════════════════════════════════╝

THREE NEW METHODS in DoomRenderer class:

1. render_wall_texture(x, y, height, width, color, pattern, shade)
   
   Purpose: Render textured walls with patterns
   
   Parameters:
     - x, y: Screen position
     - height, width: Wall dimensions
     - color: Base RGB color
     - pattern: 0=solid, 1=stripes, 2=dots, 3=grid
     - shade: Distance shading factor (0.0-1.0)
   
   Process:
     a. Apply distance shading to color
     b. Draw base colored rectangle
     c. Add pattern overlay based on type
     d. Blend with environment

2. render_wall_sign(x, y, height, width, sign_text, direction_text)
   
   Purpose: Render text signs on walls
   
   Parameters:
     - x, y: Screen position
     - height, width: Wall dimensions
     - sign_text: Main resource name
     - direction_text: Directional indicator
   
   Process:
     a. Create semi-transparent background
     b. Render resource name in yellow
     c. Render direction in green
     d. Center text on wall
     e. Truncate long text

3. render_directional_arrow(x, y, height, direction, color)
   
   Purpose: Draw directional arrows
   
   Parameters:
     - x, y: Screen position
     - height: Wall height
     - direction: "→", "←", "↑", or "↓"
     - color: Arrow color (usually green)
   
   Process:
     a. Parse direction symbol
     b. Calculate arrow position
     c. Draw triangle polygon
     d. Orient correctly
