NEW METHODS IN aws_doom.py:

1. _add_hallway(x1, y1, x2, y2, width=40)
   - Creates a corridor between two points
   - Generates parallel walls on both sides
   - Hallways are PASSABLE (no collision)
   - Dark gray color for visual distinction
   - Uses perpendicular math for proper width

2. _create_doorway(room_x, room_y, room_width, room_height, side)
   - Removes wall segments to create openings
   - Splits walls into top and bottom segments
   - Creates 60-pixel wide doorways
   - Allows passage through room walls
   - Supports 'left' and 'right' sides

NEW CONNECTIONS:

1. VPC ←→ Subnets
   - Each VPC connects to its first subnet
   - Doorway created in VPC's right wall
   - Hallway width: 50 pixels
   - Provides entry into VPC structure

2. Load Balancers ←→ VPCs
   - Each LB connects to nearest VPC
   - Doorways in both LB and VPC walls
   - Hallway width: 60 pixels
   - Represents network traffic flow

HALLWAY GENERATION ALGORITHM:

1. Calculate direction vector between two points
2. Normalize the direction
3. Calculate perpendicular direction for width
4. Create two parallel walls (left and right sides)
5. Mark walls as ResourceType.HALLWAY (passable)
6. Add to map's wall list

DOORWAY CREATION ALGORITHM:

1. Find walls of the target room on specified side
2. Calculate midpoint of the wall
3. Remove central 60-pixel segment
4. Create two wall segments (top and bottom)
5. Leave gap in middle for passage
6. Update room's wall list

COLLISION DETECTION:

  Wall Type              Passable?
  ─────────────────────  ─────────
  VPC walls              ❌ No (unless doorway)
  Subnet walls           ❌ No
  Security Group walls   ❌ No (unless authorized)
  Hallway walls          ✅ YES (always passable)
  Doorway openings       ✅ YES (gap in wall)
